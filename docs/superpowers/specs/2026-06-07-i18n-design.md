# Questline i18n Рђћ Bilingual (zh/en) Design

**Date:** 2026-06-07
**Status:** Approved by user (approach A: central locale dictionaries)

## Goal

All user-facing Questline text renders in the user's language Рђћ Chinese or
English Рђћ resolved automatically from the system locale, overridable via
config. Adding a third language later means adding one dictionary file.

## Decisions (from brainstorming)

- **Strategy:** bilingual i18n with central language packs (not full-Chinese
  rewrite, not inline ternaries, not an i18n framework).
- **Default:** follow system locale Рђћ `zh*` Рєњ Chinese, anything else Рєњ
  English. Manual override via config written by `/questline:setup`.
- **Boss names:** translated in zh (уІѓуєі, С╣Ютц┤УЏЄ, тиетЃЈ, ТђеуЂх, Тќ»УігтЁІТќ», жГћтЃЈ).
- **README:** `README.md` (English) + `README.zh-CN.md` (Chinese), cross-linked.

## Architecture

### 1. Core module Рђћ `scripts/lib/i18n.js` (~30 lines)

- `locale()` Рђћ resolution order:
  1. `locale` field in `${state.ROOT}/config.json`
  2. `LANG` / `LC_ALL` env var (`zh*` Рєњ `zh`)
  3. default `en`

  Cached per process. Hooks are short-lived processes, so staleness is not a
  concern.
- `t(key, params)` Рђћ looks up the key in the current locale's dictionary,
  interpolates `{name}`-style placeholders. Missing key Рєњ falls back to the
  English dictionary.

### 2. Language packs Рђћ `scripts/locales/en.js`, `scripts/locales/zh.js` (~45 keys each)

| Group | Contents |
|---|---|
| mapper | verb pools (9 categories ├Ќ 2РђЊ3 verbs; zh e.g. read Рєњ уфЦТјб/тІўт»Ъ, edit Рєњ ТќЕтЄ╗/уїЏтіѕ), cast line, backfire line, hit/combo line, minion-slain line |
| report | turn header, boss line, DMG line, HP line, weekly bar label, stagger prompt |
| sage | 3 advice lines |
| hud | rest banner, awaiting-encounter line, your-turn line |
| boss | type names: BugbearРєњуІѓуєі, HydraРєњС╣Ютц┤УЏЄ, ColossusРєњтиетЃЈ, WraithРєњТђеуЂх, SphinxРєњТќ»УігтЁІТќ», GolemРєњжГћтЃЈ; name template |
| commands | defeat / milestones / battlelog CLI outputs incl. error lines |

Emoji/number compounds (`­Ъњђ3 Рџћ№ИЈ42`, bars, cost ticker) are
language-neutral Рђћ not in the dictionaries.

### 3. Boss save structure (breaking-ish change, with fallback)

- `nameBoss` stores `{base, typeKey}` instead of a baked English string.
- Display renders via `t('boss.name', {base, type})` Рђћ so switching locale
  re-renders existing bosses in the new language.
- Templates: en `The ${base} ${type}`, zh `${base}┬и${type}`.
- `namer.js` (detached haiku namer): prompt asks for a Chinese boss name when
  locale is zh. AI-generated name is stored in `name` and takes precedence
  over the template.
- Old saves that only have `name`: displayed as-is (compatibility fallback).

### 4. Out of scope for the dictionaries

- `commands/*.md` are prompts to Claude; their output comes verbatim from the
  scripts, so no translation needed there.
- `setup.md` gains one step: detect/ask language, write `config.json`.

### 5. README

`README.md` stays English; new `README.zh-CN.md`; each links to the other at
the top.

## Error handling

If dictionary load or key lookup fails for any reason, fall back to
hard-coded English. A hook must never crash because of translation Рђћ this
preserves the Observer Principle (zero impact on real usage).

## Testing

- locale resolution: config override > env (`zh_CN.UTF-8` Рєњ zh, `en_US` Рєњ
  en, unset Рєњ en)
- placeholder interpolation
- zh/en key-parity check (fails if a key exists in one pack but not the
  other Рђћ guards against missed translations)
- boss name rendering: new-format save in both locales, old-format save
  fallback
