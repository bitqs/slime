# Slime v0.3 "VS 数学包" — 设计文档

日期:2026-06-11
来源:vs-anatomy 项目(吸血鬼幸存者逆向拆解,157 设计原子)对照 slime 现状的差距分析。
原则:**所有上瘾点来自数学,不来自资源堆砌。**

## 背景与目标

对照 VS 原子库,slime 当前四个最大差距:

| 差距 | VS 原子 | slime 现状 |
|------|---------|-----------|
| 升级节拍线性变慢 | ATOM-P02 升级节拍恒定 | 需求线性涨、奖励恒定 → 后期升级无感 |
| 没有开奖感 | ATOM-L01/L03/L05 | loot = 4% 平掷,无演出无期待 |
| boss 战无情绪弧 | ATOM-P14 两曲线赛跑 | HP 匀速掉,平的 |
| 无微步长养成层 | ATOM-G07 金蛋 | prestige 之外无无限成长 |

本轮一次性补齐四个系统,互相咬合:

```
击杀 ──→ XP×等级放大(①) ──→ 升级"叮"恒定节拍
  ├──→ 3% 掉蛋(④) ──→ 永久微强化 ──→ ①②③ 全变爽
  └──→ boss 死 ──→ 宝箱揭晓(②) ──→ 饰品XP + 蛋
combo ──→ 伤害乘数(③) ──→ HP 条情绪弧
```

## ① 升级节拍修复

**问题:** 升级需求 `累计 XP = 50·n·(n−1)`(每级增量线性涨),击杀 XP 基本恒定(~50-300)→ 升级间隔线性变慢。VS 的解法是两条曲线同阶赛跑:需求 ∝ L²,供给也 ∝ L²。

**方案:** 需求曲线不动(老存档零迁移),击杀 XP 随等级放大:

```
击杀 XP = 基础公式 × (1 + 0.12 × 当前等级) × prestige 乘数 × (1 + 蛋XP加成)
```

基础公式不变:`50 + min(300, ⌊40·√dmg⌋) + kills·20 + maxCombo·5`。

**节拍验证(基础 ≈259 XP/杀时):**

| 等级 | XP/杀 | 升级需求 | 杀/级 |
|------|-------|---------|-------|
| 1 | 259 | 100 | ~0.4 |
| 5 | 414 | 500 | ~1.2 |
| 10 | 570 | 1000 | ~1.8 |
| 20 | 880 | 2000 | ~2.3 |

节拍收敛到 ~2-3 杀/级,不再线性变慢。HUD 显示的击杀 XP 数字随等级变大 = 变强感(同一拳打出更大数字)。

**改动点:** `core/progression` XP 计算一处。badge XP(+100)与 quest XP(+150/+100)为里程碑奖励,保持固定,不随等级放大。

## ② 宝箱开奖系统(双舞台)

**心理学:** 老虎机三要素 — 掷骰前置保留神秘、转轮揭晓拖时间造期待、奖越大演出越长。

### 掷骰规则

- **掷定时机前置**(ATOM-L02/L03):boss 出生(spawn)时即掷定宝箱等级,存入 `BossState.chestTier`;击杀时才揭晓。玩家无从验证 → 神秘感。
- **三骰降级判定:** 依次掷 头奖(5%)→ 金(20%)→ 银(保底)。每骰独立,先中先得。
- **幸运横切**(ATOM-L04):蛋的掉落率加成同时作用于三骰判定与蛋掉落 — 一个属性处处生效。
- **新手运**(ATOM-L01):profile 记录 `chestCount`,前 6 个宝箱写死序列 `[银,银,金,银,银,头奖]`,之后回归概率。
- **确定性:** 掷骰用现有哈希方案(boss hash + chestCount 盐),replay 位级一致。

### 内容物

- 基底:现有 loot 表 XP 饰品(小/中/大),按宝箱等级上移权重。
- 夹带:银 10% / 金 30% 概率附带一颗史莱姆蛋;**头奖箱必含蛋**。

### 终端舞台(turn report)

击杀 boss 的 turn report 加揭晓行:

```
🎁 宝箱[金] → ✨大经验饰品 +80 XP · 🥚史莱姆蛋(暴击+0.5%)
```

### arena 舞台(PixiJS)

落箱 → 抖动 → 开盖光柱 → 战利品弹出。等级越高演出越长(ATOM-L05):银 ~1s,金 ~2s,头奖 ~3.5s 带全屏粒子。遵守现有 CALM 降级约束(≤3 闪/秒)。

### boss 处决仪式(ATOM-G05 白手)

- arena:boss 击杀时刻 — 慢镜(hitstop 延长)→ 闪白 → 溶解粒子 → 胜利横幅(boss 名 + 战斗统计)。
- 终端:turn report 顶部 ASCII 胜利横幅。
- **这是本轮的可分享物**:README 重录 GIF 用「处决 → 头奖开箱」连续镜头。

## ③ boss 战情绪弧

**心理学:** VS 的"被压→反超"弧是两条曲线的几何,不是脚本。slime 局内唯一的"玩家变强"变量是 combo,接进 HP 伤害即可:

```
HP 伤害 = lines × min(上限, 1 + 0.08 × combo)
上限基础 ×2(ATOM-P08 封顶清单),蛋可扩(每颗 combo蛋 +2%)
```

| combo | 乘数 | 阶段 |
|-------|------|------|
| 0-2 | ×1.0-1.16 | 前期磨血(被压) |
| 5 | ×1.4 | 起势 |
| 12+ | ×2.0 封顶 | 收割期(反超) |

- 失败一次 → combo 归零 → 回磨血期 = "连胜手热"护档张力。
- **仅作用于 HP 条观感层。** XP 计算、防农逻辑(test-kill 签名、ULTIMATE blocker)零改动。
- arena 伤害数字随乘数分档变大变红(ATOM-F06)。

## ④ 史莱姆蛋(无限成长层)

**心理学**(ATOM-G07 金蛋):步长极小(百颗成神)、属性随机(每颗都是小开奖)、永久累积(长期投入)。

### 掉落

- 每次击杀 3% 掉(蛋掉落加成横切),宝箱按 ② 规则夹带。
- 确定性哈希掷骰(kill 事件计数盐),replay 稳定。

### 属性池(权重随机一项)

| 属性 | 步长 | 权重 |
|------|------|------|
| ✨ XP 增益 | +1% | 40 |
| 🎁 掉落率 | +0.2% | 30 |
| 💥 暴击率(arena PRD 基础) | +0.5% | 25 |
| 🔥 combo 伤害上限 | +2% | 5(稀有) |

### 存储与展示

- `profile.eggs = { xp: n, loot: n, crit: n, combo: n }`,**prestige 不清零**。
- HUD 进度段加 `🥚×总数`;`/slime:achievements` 显示分项明细。
- 掉蛋时 turn report 一行:`🥚 史莱姆蛋!✨XP增益 +1%(累计 ×1.23)`。

## 兼容与迁移

- 老 profile 无 `eggs`/`chestCount` 字段 → 读取时默认补 0,无迁移脚本。
- 需求曲线、等级、称号、badge、quest、prestige 规则全部不动。
- 防农三件套(test-kill 签名、ULTIMATE blocker、伤害 sub-linear 封顶)原样保留。

## 测试

公式层全为纯函数,单测覆盖:

1. 节拍收敛:L1→L25 模拟,杀/级单调收敛到 2-3 区间。
2. 三骰分布:10k 次掷骰统计 ≈ 5%/19%/76%(头奖/金/银),幸运加成方向正确。
3. 新手运:前 6 箱严格输出写死序列,第 7 箱起走概率。
4. 蛋权重:10k 次掷骰 ≈ 40/30/25/5。
5. combo 封顶:乘数不超 2×(1+蛋加成),combo 归零回 ×1。
6. 老存档读取:缺字段补默认,不抛错。
7. 确定性:同输入序列两次 replay,掷骰结果位级一致。

## 不做(YAGNI)

- 不做属性面板 UI(蛋明细放 achievements 即可)。
- 不做宝箱"留箱择机开"机制(掷骰前置已封死,也无此需求)。
- 不做项目绑定养成(用户项目生命周期短,投入易作废)。
- 不动 quest/badge 体系。
